# Campsoon 用户使用指南

> 图片占位：在这里放一张 Campsoon 扩展主界面的截图。

Campsoon 是一个帮助用户自动关注 BC Parks 营地空位的工具。你可以设置想去的营地和日期，系统会持续扫描空位；当发现符合条件的营地时，系统会尽快帮你进入预订流程。

## 重要概念

### 账号

使用 Campsoon 前需要先登录账号。账号用于保存你的行程设置、点数余额和预订结果，也方便系统在预订成功或失败时通知你。

### Trip

Trip 是一次营地搜索任务。你可以把它理解成“我想订哪几个营地、哪几天、有什么筛选条件”的组合。

一个 Trip 通常包括：

- 想去的营地
- 入住和离开日期
- 是否排除 walk-in 营地
- 是否排除 double site
- 发现空位后的处理模式

### 营地优先级

如果你在一个 Trip 里添加了多个营地，系统会按照你设置的顺序扫描。排在前面的营地优先级更高。

例如你把 Alice Lake 放在第一位、Cultus Lake 放在第二位，系统会优先检查 Alice Lake。

### 扫描

扫描就是系统定期检查 BC Parks 是否有新的空位。热门营地的空位可能很快被别人抢走，所以扫描需要持续进行。

### Match

Match 是指系统发现了符合你 Trip 条件的空位。发现 match 后，Campsoon 会根据你选择的模式继续处理。

### 三种处理模式

Campsoon 支持三种模式：Alert Only、Auto-reserve 和 Auto-pay。你可以根据自己想要的自动化程度选择。

#### Alert Only

Alert Only 是免费功能。系统发现空位后，会给你发送邮件提醒，但不会帮你预订，也不会付款。

适合只想收到空位通知、然后自己手动进入 BC Parks 操作的用户。

#### Auto-reserve

Auto-reserve 会在发现空位后，自动帮你尝试 Reserve，并把营地加入购物车。

成功 Reserve 后，你会收到邮件提醒。你只需要回到已经打开的 BC Parks 标签页，确认信息并点击支付即可。

这种模式适合你不确定系统找到的营地是否完全符合偏好，需要二次确认后再决定是否付款的情况。使用时建议你在电脑旁边，方便及时完成付款。

#### Auto-pay

Auto-pay 会在发现空位后，自动帮你完成预订和付款，整个过程不需要你手动操作。预订成功后，你也会收到邮件提醒。

使用 Auto-pay 前，你需要在 Campsoon 的 Park Payment 页面填写信用卡号、持卡人姓名、有效期、CVV 和账单地址等信息。

这些信用卡信息会完全加密存储在你的本地浏览器扩展中，不会上传到 Campsoon 服务器。Campsoon 服务器不会记录任何信用卡相关信息。

盗用信用卡是违法犯罪行为。为了进一步降低风险，如果你仍然不放心，建议使用预付费信用卡进行支付。

适合目标非常明确、希望系统自动完成全部流程，或者不方便一直守在电脑旁边的用户。由于 BC Parks 空位变化很快，即使使用 Auto-pay，也不能保证 100% 预订成功。

### 点数

点数是 Campsoon 的使用额度。你可以购买点数，系统会在成功帮你完成一次营地预订后扣除相应点数。

没有成功预订时，一般不会因为单纯扫描而扣除成功预订费用。

## 手动安装 Chrome 插件

如果你收到的是 Campsoon 插件压缩包，可以按照下面步骤手动安装：

1. 先把压缩包解压到一个固定位置，例如桌面或 Documents 文件夹。
2. 打开 Chrome，在地址栏输入 `chrome://extensions/` 并回车。
3. 打开右上角的「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择刚刚解压出来的 Campsoon 插件文件夹。
6. 安装完成后，点击 Chrome 右上角的扩展图标，打开 Campsoon。

## 使用流程

### 1. 安装并打开 Campsoon

安装 Chrome 扩展后，点击浏览器右上角的 Campsoon 图标打开工具。

> 图片占位：在这里放一张 Chrome 扩展入口或登录页截图。

### 2. 登录账号

输入你的邮箱并完成验证码登录。登录后，你可以看到自己的账号状态和点数余额。

### 3. 创建 Trip

新建一个 Trip，然后填写你想要的营地和日期。

建议一开始不要设置太多条件。条件越多，找到合适空位的机会越低。

> 图片占位：在这里放一张创建 Trip 的截图。

### 4. 设置营地和日期

添加你想去的营地，并按照自己的偏好排序。最想去的营地放在前面。

然后添加可以接受的入住日期。如果你的时间比较灵活，可以添加多个日期，提高成功机会。

### 5. 选择处理模式

选择发现空位后的处理方式：

- Alert Only：只提醒，不预订。
- Auto-reserve：帮你尝试保留位置，付款由你手动完成。
- Auto-pay：帮你尝试完成预订和付款。

如果选择 Auto-pay，请先到 Park Payment 页面添加信用卡信息。

> 图片占位：在这里放一张选择 Trip 模式的截图。

### 6. 检查筛选条件

你可以选择是否排除 walk-in 营地或 double site。

- Walk-in 营地通常需要步行进入，不适合所有用户。
- Double site 通常是较大的组合营位，费用和规则可能不同。

如果你不确定，建议先保留默认设置。

### 7. 开始扫描

确认 Trip 信息无误后，启动扫描。系统会在后台持续检查空位。

扫描期间需要保持 Chrome 浏览器和 Campsoon 插件正常运行，并避免频繁修改同一个 Trip。

请不要让电脑进入休眠状态。如果电脑休眠、关机，或 Chrome 被关闭，插件就无法继续扫描，也无法自动 Reserve 或 Auto-pay。

> 图片占位：在这里放一张 Trip 正在扫描的截图。

### 8. 等待结果

如果系统发现符合条件的空位，会根据你选择的模式提醒、保留或付款。预订成功后，你会看到结果通知，并可以根据提示查看订单信息。

如果预订失败，常见原因是空位已经被别人抢走、BC Parks 页面变化、付款信息异常，或网络连接不稳定。

## 收费和点数

Alert Only 是免费功能，只发送空位邮件提醒。Auto-reserve 和 Auto-pay 是收费功能，只有成功完成预订后才会扣除点数，每次成功预订扣除 1000 points。现在的点数套餐是：

- 1000 points：CAD 10
- 3000 points：CAD 20

收费主要是为了避免无限制扫描影响大家抢营地的成功率，同时覆盖开发、服务器和维护成本，让服务可以长期稳定运行。

## 使用建议

- 优先选择你真正愿意去的营地，不要一次性添加太多不相关目标。
- 日期越灵活，成功机会越高。
- 使用 Auto-pay 前，请确认你的 BC Parks 账号、付款信息和出行计划都准备好。
- 如果一个 Trip 长时间没有结果，可以尝试放宽日期或增加备选营地。
- 预订成功后，请及时查看 BC Parks 的确认信息。

## 常见问题

### 发现空位后一定能订到吗？

不能保证。Campsoon 会尽快尝试预订，但热门空位可能在几秒或几十秒内被其他人抢走。

### 扫描会一直运行吗？

只要你的 Trip 处于扫描状态，系统会按一定频率检查空位。你也可以随时暂停或停止 Trip。

### 为什么我需要登录？

登录可以帮助系统保存你的 Trip、同步点数余额、记录预订结果，并确保每个账号的使用都符合公平规则。

### 为什么要用点数？

点数可以让使用成本更透明，也能避免无限制扫描。通常只有成功完成预订后才会扣除对应点数。

### 我的信用卡信息会上传到服务器吗？

不会。Park Payment 里的信用卡信息会完全加密存储在你的本地浏览器扩展中，不会上传到 Campsoon 服务器，服务器也不会记录任何信用卡相关信息。如果你希望把风险降到最低，可以使用预付费信用卡。

### 如果预订失败怎么办？

你可以检查 Trip 设置、付款信息和网络状态，然后重新开始扫描。如果是空位被别人抢走，可以继续等待下一个空位。

## 重要提醒

使用 Campsoon 扫描时，需要保持电脑开机、Chrome 浏览器打开，并确保 Campsoon 插件正在运行。电脑休眠后，插件不会继续工作。

Campsoon 是辅助工具，不能保证一定订到营地。最终预订结果以 BC Parks 的确认信息为准。请在出行前仔细核对日期、营地、人数、车辆和付款信息。
